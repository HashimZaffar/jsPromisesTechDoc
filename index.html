Introduction

JavaScript Promises are a powerful way to handle asynchronous operations. Promises simplify code, improve readability, and make it easier to manage async tasks.

Creating Promises

Promises are created using the Promise constructor. A promise represents an operation that hasn't completed yet, but is expected in the future.


const myPromise = new Promise((resolve, reject) => {
    // Async operation here
    if (operationSuccess) {
        resolve('Operation successful');
    } else {
        reject('Operation failed');
    }
});

Consuming Promises

Promises can be consumed using the .then() and .catch() methods.


myPromise.then((result) => {
    console.log(result);
}).catch((error) => {
    console.error(error);
});

Chaining Promises

Promises can be chained together to handle sequences of asynchronous operations.


myPromise.then((result) => {
    return anotherAsyncOperation(result);
}).then((finalResult) => {
    console.log(finalResult);
}).catch((error) => {
    console.error(error);
});

Error Handling

Errors in promises can be handled using the .catch() method, which catches any errors that occur in the promise chain.


myPromise.catch((error) => {
    console.error('An error occurred:', error);
});

Conclusion

Promises are an essential tool in JavaScript for managing asynchronous operations. By understanding how to create and consume promises, you can write more efficient and maintainable code.
